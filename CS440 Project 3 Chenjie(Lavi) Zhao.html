Problem Definition

The goal of this project is to implement an algorithm to win a 4x4x4 tic-ta-toe game using minimax and alpha-beta pruning technique. Minimax algorithm needs to be implemented to provide the next move and alpha-beta pruning is implemented to reduce the game tree or search time. This project illustrates how minimax algorithm applies to a real life situation.

I assumed the rules the AI has to follow, in the order of importance, are:

“Rule 1: If the maximizing player has a winning move, take it.

Rule 2: If the minimizing player has a winning move, block it.

Rule 3: If the maximizing player can create two winning lines after this move, do it.”

The anticipated difficulty is figuring out a correct way to evaluate the board and set all the heuristic values for the game tree so that the algorithm will do exactly what the rules above dictates.

Experiments

Experiment 1: AI vs. Random Move Generator

I modified the runTicTacToe class so that player 1 uses the AI algorithm I implemented with depth = 2 and player 2 uses the random move generator that was provided to us in the skeleton code. Which player goes first is randomized. I repeated the game loop for 1000 times for each trial. I then evaluated the winning rates and efficiency for making a decision of player 1 under 3 trials. (Depth = 2 is used because it is sufficient enough to beat the random generator and uses significantly less time to make a decision)

Experiment 2: AI vs. AI under different depth and same heuristic function

Discussion

From the experiment results, we can see that the method is very successful competing a random move generator as expected. On average, the algorithm with a depth of 2 was sufficient to beat the random generator within 10 moves. The algorithm is successful competing against a version of itself with less depth as well. As we can see that the competition always results in a first player win as expected, since the first player always does a deeper search than the second player.

Strengths:

  • Uses alpha-beta pruning to optimize running time
  • Great performance against random move generators

Weaknesses:

  • Very static response for a specific board

Future Work:

  • Improve the algorithm so it detects potential fork(multiple winning line) in future moves
  • Experiment with more different heuristic values and depths to find the perfect combination

Conclusions

Game theory can be a substantial part of AI development. In this programming assignment, we experimented with minimax algorithm using alpha-beta pruning. As we can see, with a depth of 2, the algorithm is already very successful with random move generator. However, more work can be done for it to compete with another algorithm, such as experiment with more different heuristic values and depths to find the perfect combination.
